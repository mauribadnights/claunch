Features

Everything you need to manage agents across projects, nothing you don't.

▷

Split-Panel TUI

Interactive two-phase picker: choose an agent on the left, then a target directory on the right. Animated Claude logo with breathing effect.

⌕

Fuzzy Search

Type to instantly filter agents and directories. Consecutive-character bonus scoring finds what you want in 2-3 keystrokes.

★

Frecency Ranking

Mozilla-style exponential decay with a 14-day half-life. Your most-used agents float to the top automatically.

⚙

Auto-Discovery

Run claunch scan and it walks your directories, finding projects by markers like .claude/agents/, package.json, and CLAUDE.md.

⇄

Agent Overrides

Override working directories, add --add-dir flags, or remap agent names per-project. Full control when you need it.

⌨

Shell Completions

First-class tab completion for zsh, bash, and fish. Project names and agent names complete inline.

Quickstart

Up and running in under a minute.

1

Install

npm install -g @mauribadnights/claunch
2

Register projects

Point claunch at directories containing .claude/agents/.

claunch add myapp ~/projects/myapp
claunch add infra ~/infrastructure

Or auto-discover everything under a root:

claunch scan ~/projects
3

Launch

Interactive mode or direct invocation.

# Interactive TUI
claunch

# Direct launch
claunch myapp cto

# Plain claude in a project dir
claunch myapp plain
4

Add shell completions (optional)

# zsh
eval "$(claunch completions zsh)"

# bash
eval "$(claunch completions bash)"

# fish
claunch completions fish > ~/.config/fish/completions/claunch.fish

Configuration

Everything lives in ~/.claunch/config.yaml

~/.claunch/config.yaml
# Global defaults
defaults:
  claude_flags: []

# Roots for `claunch scan` auto-discovery
scan_roots:
  - ~/projects
  - ~/work

# Registered projects
projects:
  myapp:
    dir: ~/projects/myapp

  infra:
    dir: ~/infrastructure
    agents_dir: ~/shared-agents  # custom agent path

  monorepo:
    dir: ~/projects/monorepo
    overrides:
      cto-core:
        dir: ~/projects/monorepo/packages/core
        add_dirs:
          - ~/projects/monorepo/docs
        agent: cto  # remap to the cto agent definition

CLI Reference

claunch Interactive project/agent picker (TUI)
claunch <project> List agents for a project
claunch <project> <agent> [args] Launch an agent in project context
claunch add <name> <dir> Register a project directory
claunch remove <name> Unregister a project
claunch scan [root-dir] Auto-discover projects from directory tree
claunch list List all projects (non-interactive, pipe-safe)
claunch init Create default config at ~/.claunch/config.yaml
claunch completions <shell> Print completions for zsh, bash, or fish
claunch <project> plain Launch plain Claude (no agent) in project dir

How It Works

claunch reads from what you already have. No new config format to learn.

your project structure
~/projects/myapp/
|
+-- .claude/
|   +-- agents/
|       +-- cto.md          <-- claunch discovers these
|       +-- designer.md
|       +-- devops.md
|
+-- src/
+-- package.json

When you run claunch myapp cto, it cds to the project directory and runs claude --agent cto. That's it.
